Attention-routing fidelity as a self-modification signal

An intrinsic attention signal let the agent rewrite its own code — but couldn't tell it the code had gone dead.

Study 001's agent never touched Python and drifted downhill with no feedback. Study 002 gave it one internal signal — how much of the model's attention lands on results sentences — in place of any score. The signal unlocked code changes. It could not guide them.

The evidence, in one picture

Extraction quality falls off a cliff while the signal the agent steered by barely moves.

The two series are drawn on separate axes on purpose — their scales are 25× apart, and a shared dual axis would fake a relationship that isn't there. At iteration 6 the agent's two-call refactor produced zero claims on every abstract. The routing score registered the catastrophe as a shrug, then flatlined.

Why the signal went blind

Routing measures attention over the model's input — not the health of the pipeline that consumes it.

This run used the corrected instrumentation (amendment A009-1), so attention was measured over the extractor's real first model call, exactly as pre-registered. It still didn't move when the agent edited code — because the edits changed downstream parsing and pipeline logic, not the text handed to that first call.

Direct evidence · routing_history.jsonl
Iteration 4 was an extractor.py-only edit. Its routing score is byte-identical to iteration 3's — 0.009366872215545337, delta 0.0. Iteration 9's code edit: also delta 0.0. A signal watching input attention cannot see a pipeline change underneath it.

The bug (measuring the wrong call) was real and is fixed. The residual decoupling — watching input attention rather than the pipeline's behavior — is a property of the signal's design, and it survives the fix. That is why a fully dead extractor read "flat" for seven straight iterations.

Bad execution, or bad strategy?

Even implemented correctly, the agent's refactor loses to leaving the extractor alone.

The iteration-6 pipeline failed because the agent's own prompt contradicted its own injection. But an offline replication that implemented the same two-stage decomposition correctly still scored far below the untouched baseline on iteration 6's exact draw — both precision and recall collapse. The harness bugs decided how catastrophically the run failed, not whether the direction could have won. It couldn't.

Threats to validity

What the study establishes, and what the instrumentation confounds.

The negative result is partly entangled with harness defects found and amended across implementation (series A004–A009). The core finding — an input-attention signal is decoupled from extraction quality — is a design property, not a bug. But three instrumentation gaps shaped how the run failed.

§5.2Context truncation suppressed the fixes

Prompts grew to 7–8.7k tokens against a 5,120-token budget; truncation kept the file tail and discarded the agent's edit blocks. It repeatedly diagnosed its own bug, then had the fix cut off (no_edits_proposed at iterations 7, 8, 10, 11, 12).

§5.3No dead-extractor guard

The smoke test passed at iteration 6 with a claim count of zero. No breaker halts ~100% zero-claim output, so a dead extractor ran for seven iterations without stopping itself.

§5.4Signal near its own floor

Routing values stay under ~2.5% and fall to ~0.6% in the dead tail — plausibly dominated by noise. The study cannot fully separate "the agent failed to use a real signal" from "there was no real signal to use."

What it means for the program

A constraint on intrinsic signals, not a dead end.

Study 002 delivers two usable results. The behavioral one — plain capability framing produces code changes — is a clean positive that will be held constant, so later studies inherit an agent that actually uses its full modification surface. The signal one is a bounded negative: attention-over-input routing fidelity is not a sufficient intrinsic cost signal. A useful intrinsic signal likely has to be computed over the model's output-generating pass or the pipeline's behavior — not its input attention. Study 003 proceeds with the baseline correction retained and the intrinsic-signal question sharpened by what routing fidelity could not do.
